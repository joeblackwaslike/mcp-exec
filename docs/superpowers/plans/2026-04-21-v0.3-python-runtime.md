# v0.3 Python Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a stateless Python runtime to `exec()` via `uv run --isolated`, with PEP 723 inline dependency support and a `py-sdk-reference.md` skill reference.

**Architecture:** Python code is written to a temp file, executed via `uv run --isolated <tmpfile>`, stdout captured as `result`, temp file deleted unconditionally in a `finally` block. Mirrors `bash.ts` exactly in shape. No sessions, no MCP client injection.

**Tech Stack:** Node.js 22 LTS, Python 3.12.x, `uv` (must be installed on PATH), Vitest for tests.

```typescript
// src/sandbox/runtimes/python.ts
import { spawn } from 'node:child_process';
import { randomUUID } from 'node:crypto';
import { unlink, writeFile } from 'node:fs/promises';
import { tmpdir } from 'node:os';
import { join } from 'node:path';
import type { ExecResult } from '../../types.js';

/** Runs Python code in a stateless uv subprocess. No session state persists between calls. */
export async function runInPython(
  code: string,
  opts: { timeout?: number; env?: Record<string, string> } = {},
): Promise<ExecResult> {
  const tmpFile = join(tmpdir(), `mcp-exec-${randomUUID()}.py`);
  await writeFile(tmpFile, code, 'utf8');

  try {
    return await spawnPython(tmpFile, opts);
  } finally {
    await unlink(tmpFile).catch(() => {});
  }
}

function spawnPython(
  scriptPath: string,
  opts: { timeout?: number; env?: Record<string, string> },
): Promise<ExecResult> {
  return new Promise((resolve) => {
    const stdoutChunks: Buffer[] = [];
    const stderrChunks: Buffer[] = [];

    const child = spawn('uv', ['run', '--isolated', scriptPath], {
      env: { ...process.env, ...opts.env },
      stdio: ['ignore', 'pipe', 'pipe'],
    });

    let timedOut = false;
    const timer = opts.timeout
      ? setTimeout(() => {
          timedOut = true;
          child.kill('SIGTERM');
        }, opts.timeout)
      : null;

    child.stdout.on('data', (chunk: Buffer) => stdoutChunks.push(chunk));
    child.stderr.on('data', (chunk: Buffer) => stderrChunks.push(chunk));

    child.on('close', (code) => {
      if (timer) clearTimeout(timer);
      const stdout = Buffer.concat(stdoutChunks).toString();
      const stderr = Buffer.concat(stderrChunks).toString();
      const exitCode = timedOut ? 124 : (code ?? 1);

      resolve({
        result: stdout,
        stdout,
        stderr,
        exitCode,
        tool_calls: [],
      });
    });
  });
}
```

# Python MCP SDK Reference for mcp-exec

> Pre-processed reference for writing Python code inside `exec()` calls.
> Python runs stateless â€” no session state persists between calls.

## Runtime environment

- **Python version:** 3.12.x (via uv)
- **Isolation:** `uv run --isolated` â€” no access to the user's installed packages
- **State:** stateless â€” each `exec()` call is a fresh process
- **Result channel:** stdout only â€” `print()` is the equivalent of `return` in Node

---

## Declaring dependencies with PEP 723

Add an inline script header to declare dependencies and Python version. `uv` parses
this natively and installs into a throwaway environment:

```python
# /// script
# requires-python = ">=3.12"
# dependencies = ["pandas>=2.0", "httpx"]
# ///

import pandas as pd
import httpx

# your code here
print("result")
```

**Always include this header when using third-party packages.** Without it, only the
standard library is available.

uv caches resolved environments by dependency hash â€” first use of a dependency set is
slow (~2-10s); subsequent calls with identical deps are fast (<100ms overhead).

## stdout as the return channel

Only stdout enters the context window. Everything you want Claude to see must be printed:

```python
import json, sys

data = {"status": "ok", "count": 42}
print(json.dumps(data))   # this reaches Claude
```

`stderr` (tracebacks, warnings) is captured separately and appears in the `stderr` field
of the exec result â€” visible for debugging but not in `result`.

---

## Receiving data from a Node exec

Python has no access to Node session state. Thread data by writing it to a temp file in
Node and reading it in Python, or pass small payloads via env vars.

**Temp file pattern (for large payloads):**

```typescript
// exec 1 (node) â€” write data to a temp file
import { writeFileSync } from 'fs';
const data = await fetchLargeDataset();
writeFileSync('/tmp/mcp-exec-data.json', JSON.stringify(data));
return '/tmp/mcp-exec-data.json';
```

```python
# exec 2 (python) â€” read from the temp file
# /// script
# dependencies = ["pandas"]
# ///
import pandas as pd, json

with open('/tmp/mcp-exec-data.json') as f:
    data = json.load(f)

df = pd.DataFrame(data)
print(df.describe().to_json())
```

**Env var pattern (for small payloads):**

```typescript
// exec 1 (node)
const value = await getThing();
return value; // pass as env var in exec 2
```

```typescript
// orchestrating code â€” pass node result as env var
exec({ runtime: 'python', code: `...`, env: { INPUT: nodeResult.result } })
```

```python
# exec 2 (python)
import os
value = os.environ['INPUT']
print(value.upper())
```

## Error handling

A non-zero exit code surfaces as `exitCode` in the exec result. The Python traceback
appears in `stderr`. Claude can read `stderr` and retry with corrected code.

```python
# raise naturally â€” mcp-exec captures the traceback in stderr
df = pd.read_csv('/tmp/missing.csv')   # FileNotFoundError â†’ exitCode=1, traceback in stderr
```

---

## Limitations

- **No MCP tool calls** â€” Python cannot call `__mcpClients`. Use Node for MCP tool
  orchestration, then thread results to Python for processing. Tracked: `mcp-exec-ij3`
- **No session state** â€” `globalThis` is Node-only. Each Python exec is a fresh process.
  Tracked: `mcp-exec-1sb`
- **No stdin** â€” the Python process stdin is not connected. Use files or env vars to
  pass data in.

---

## exec result shape

```python
# mcp-exec returns to Claude after exec({ runtime: 'python', ... }):
{
  "result": "<everything printed to stdout>",
  "tool_calls": []   # always empty for Python
}

# On runtime error (non-zero exit):
{
  "result": "",      # stdout up to the error point
  "tool_calls": []
}
# stderr contains the traceback
```
