# Design: v0.3 — Python Runtime

**Status:** Draft  
**Date:** 2026-04-21  
**Scope:** Python runtime via `uv run --isolated`, `py-sdk-reference.md`, pinned runtime versions, curated package documentation

---

## What we're building

v0.3 adds a stateless Python runtime to `exec()`. It follows the same shape as the Bash runtime: each call spawns an isolated subprocess, captures stdout as `result`, and returns. No session state persists between calls.

The primary use case is data processing — pandas transforms, CSV analysis, structured output generation — typically as a downstream step after Node has fetched raw data via MCP tools.

---

## What we're explicitly not building

- **Python sessions** — deferred to the Runtime interface refactor (`mcp-exec-1sb`)
- **MCP tool calling from Python** — requires an IPC/HTTP proxy bridge; tracked as RFC in `mcp-exec-ij3`
- **Pre-warmed package environments** — uv's cache makes first-run latency a one-time cost; documented curated sets are sufficient

---

## Architecture

### Execution model

Python code is written to a temporary file, executed via `uv run --isolated`, and the file is deleted after the process exits. `python -c` is not used because PEP 723 inline metadata must appear in a file, not on the command line.

```
exec({ runtime: "python", code }) →
  write /tmp/mcp-exec-<uuid>.py →
  uv run --isolated /tmp/mcp-exec-<uuid>.py →
  capture stdout/stderr →
  delete tempfile →
  return ExecResult
```

`result` is stdout. The caller (Claude) is responsible for threading data between runtimes explicitly — Python has no access to Node session state or `__mcpClients`.

### Dependency and version management via PEP 723

Dependencies and Python version are declared inline in the script header. `uv run` parses this natively:

```python
# /// script
# requires-python = ">=3.12"
# dependencies = ["pandas>=2.0", "httpx"]
# ///

import pandas as pd
import sys, json

data = json.loads(sys.stdin.read())  # receive from Node via stdout threading
df = pd.DataFrame(data)
print(df.describe().to_json())
```

If no PEP 723 header is present, `uv` uses its default Python version (3.12.x) with no additional packages. Claude should always include the header when using third-party packages.

uv caches resolved environments by dependency hash — first-run installs are slow (~2-10s depending on packages); subsequent calls with the same deps are fast (cache hit, <100ms overhead).

Data from a preceding Node exec is threaded to Python by writing it to a temp file in Node and reading it in Python, or by passing it as an env var for small payloads. The Python process's stdin is not connected to any prior exec output.

### Temp file handling

- Path: `os.tmpdir()/mcp-exec-<randomUUID()>.py`
- Written with `fs.writeFile` before spawn, deleted in a `finally` block after process exit
- On timeout or error, the file is still deleted — cleanup is unconditional

### Timeout and exit code

Mirrors `bash.ts` exactly:
- Default timeout: none (inherits caller's setting)
- On timeout: `SIGTERM`, exit code 124
- On non-zero exit: `exitCode` reflects the process exit code, `stderr` contains the traceback

### Type system changes

`RuntimeParam` gains `'python'` and `{ type: 'python', ... }`:

```typescript
export type RuntimeParam =
  | 'node'
  | 'bash'
  | 'python'
  | { type: 'node' | 'bash' | 'python'; timeout?: number; env?: Record<string, string> };
```

The `exec` tool's MCP schema enum gains `'python'` in both the string shorthand and the object `type` field.

## Runtime asymmetry

Python and Bash are stateless subprocesses; Node has sessions and `__mcpClients`. This asymmetry is intentional for v0.3 — each runtime has a clear purpose:

- **Node**: MCP tool orchestration, session state accumulation
- **Python**: data processing, analysis, structured output generation  
- **Bash**: system automation, file operations, shell pipelines

Two issues track closing the asymmetry:
- `mcp-exec-1sb` — Runtime interface for stateful Python/Bash sessions
- `mcp-exec-ij3` — RFC: HTTP/SSE proxy for MCP tool calls from all runtimes
